## 1. ファネル全体図と離脱予測

```
[App Store 表示]  100,000 imp
      ↓ CVR 3%
[ダウンロード]     3,000 DL
      ↓ CVR 60%
[オンボーディング完了]  1,800人
      ↓ CVR 70%
[初回チャット完了]  1,260人
      ↓ CVR 40%（7日間）
[7日後リテンション]   504人
      ↓ CVR 15%
[Pro 課金]          76人/月
```

**最大の「詰まり箇所（Blocked Artery）」は3箇所：**

| ボトルネック | 離脱率 | 主因 |
|---|---|---|
| ① App Store → DL | 97% | 価値提案が5秒で伝わらない |
| ② 初回チャット → 7日リテンション | 60% | 「続ける理由」がない |
| ③ リテンション → 課金 | 85% | 無料で十分と感じてしまう |

## 2. O/CO（異議 × 反論）テーブル

### ダウンロード前の壁

| 異議（Objection） | 反論（Counter-Objection） | 配置場所 |
|---|---|---|
| 「AIに話しかけるのは恥ずかしい」 | 「誰にも知られない。履歴はあなただけのもの」 | App Store 説明文1行目 |
| 「どうせ的外れな返答が来る」 | 「感情の整理が目的。正解は出さない」 | スクリーンショット2枚目 |
| 「個人情報が心配」 | 「名前・メアド不要。匿名で即スタート」 | App Store 説明文 |
| 「どんなアプリか想像できない」 | 実際の会話スクリーンショットを見せる | App Storeプレビュー動画 |

### オンボーディング中の壁

| 異議 | 反論 | 配置場所 |
|---|---|---|
| 「登録が面倒そう」 | 「ボタン1つで即開始」← 実際にそうする | 起動直後の画面 |
| 「何を話せばいいかわからない」 | 話題セレクター（仕事/人間関係/将来/なんとなく） | 最初の画面 |
| 「読まれそうで怖い」 | 「会話ログは端末内に暗号化保存」を明示 | オンボーディング画面3 |

### 課金の壁

| 異議 | 反論 | 配置場所 |
|---|---|---|
| 「無料で十分」 | 制限に当たった瞬間だけ表示（強制ではなく文脈） | 5通目の返信後 |
| 「高い」 | 「コーヒー1杯分で、月いつでも話せる」 | Paywall画面 |
| 「続けられなそう」 | 「いつでも解約OK。1タップで止まる」 | Paywall画面下部 |

## 3. 説得資産（Persuasion Assets）の設計

### 今すぐ使えるもの（ローンチ時から）
- **匿名性の視覚化** → 鍵のアイコン + 「サーバーに名前は保存されません」
- **即開始の摩擦ゼロ** → 「Apple ID不要」バッジをApp Storeに明記
- **会話サンプル** → リアルに近いスクリーンショット（作り込みすぎない）

### 3ヶ月後に取得するもの
- ユーザーの感想（App Storeレビュー）→ Paywallに引用
- 「3日継続率○%」などの実数 → 数値が丸くないほど信頼される
- メディア掲載・SNS投稿 → App Storeスクリーンショットに追加

### 数値の見せ方の鉄則
```
NG: 「多くのユーザーが効果を実感」
OK: 「開始3日で、1,247人が『話して楽になった』と回答」
```

## 4. App Store A/Bテスト（優先度順）

### テスト① アイコン（ICE: 9.0）
```
A案: 深い紺色の背景 + 白い炎のアイコン（「内側の熱さ」）
B案: 暗いグラデーション + 余白多めの白い半円（「静けさ・安心」）
→ 推奨: B案。ターゲットは「落ち着きたい人」
```

### テスト② 説明文1行目（ICE: 8.5）
```
A案: 「誰にも言えないことを、AIと話そう」
B案: 「夜、誰かに話を聞いてほしいとき」
C案: 「本音を言える場所が、スマホの中に」
→ 推奨: B案。具体的な場面を想起させる
```

### テスト③ オンボーディング開始画面（ICE: 8.0）
```
A案: 「今日、どんな気持ちですか？」+ 感情4択
B案: 「何があったか、話してみませんか？」+ 即チャット開始
→ 推奨: B案。摩擦を限界まで減らす
```

### テスト④ Paywallのフレーミング（ICE: 7.5）
```
A案: 機能一覧型「無制限チャット・週次レポート・感情カレンダー」
B案: 感情訴求型「毎日話せる場所を、手放さないで」
→ 推奨: B案。機能より感情的な価値を売る
```

## 5. UI設計の核心決断

### 「何を捨てるか」

UI/UX Pro Max の原則に従い、**削る決断**を先にする：

```
捨てるもの:
× ホーム画面のグラフ（最初は要らない、続けてから価値が出る）
× タブバー（v1は画面遷移をシンプルに。チャットだけでいい）
× プロフィール設定（匿名なので不要）
× 通知設定画面（シンプルにオン/オフだけ）

残すもの:
○ チャット画面（アプリの95%の価値がここ）
○ 話題セレクター（最初の摩擦を消す）
○ セッション終了後の感情タグ確認
○ 週次レポート（課金の動機になる唯一の画面）
```

### 主要4画面の設計方針

---

#### 画面① オンボーディング（主役：「今夜、話してみよう」という決断）

```
背景: #0D0D1A（深い夜を想起させる濃紺）
文字: white/90

構成（3画面のみ。4画面以上は捨てる）:

[1/3] 大きな余白 + 一文だけ
  「夜、誰かに話を聞いてほしくなることはありますか？」
  ↓ tap anywhere to continue

[2/3] 安心の担保
  🔒  名前もメアドも不要
  🔒  会話は端末に暗号化保存
  🔒  AIはあなたを批判しない
  ↓ [はじめる]ボタン（画面下部・大きく）

[3/3] 即チャット画面へ（登録画面なし）
```

**批判:** 多くのアプリは3画面に機能説明を詰め込む。honneは「何ができるか」ではなく「どんな気持ちになれるか」だけを伝える。

---

#### 画面② チャット（主役：AIの「返答の質感」）

```
背景: #0D0D1A
AIバブル: bg-white/8, border border-white/10, blur(20px) → glassmorphism
ユーザーバブル: bg-indigo-600/80

最重要UXルール:
1. AIの返答は「.」→「..」→「...」でタイプ感を演出（即表示しない）
2. 返答が来たら画面が静かに下スクロール
3. 入力欄はシンプルなテキストのみ。送信ボタンは矢印1つ
4. キーボードが出たとき、AIのメッセージが隠れないように

削除するもの:
× スタンプ・リアクション機能
× 写真添付
× 音声再生ボタン（v1では不要）
```

**テキストのコントラスト（アクセシビリティ）:**
```
AIメッセージ本文:  text-white/85  → コントラスト比 ≥ 7:1 ✓
ユーザーメッセージ: text-white     → コントラスト比 21:1  ✓
タイムスタンプ:    text-white/50  → 補足情報なのでOK
```

---

#### 画面③ ホーム（主役：「連続記録」のストリーク）

```
シンプルに1つだけ見せる:

  ╔══════════════════════╗
  ║   🔥 7日連続         ║  ← これだけ大きく
  ║   話し続けています    ║
  ╚══════════════════════╝

  今週の感情:  😔 疲れ  😤 イライラ  😌 落ち着き

  [今日も話す] ← 大きなCTAボタン

削除するもの:
× 感情カレンダー（v1は不要。v1.1以降）
× 統計グラフ（データが溜まるまで空白が悲しい）
× ニュースフィード的な要素
```

---

#### 画面④ Paywall（主役：「無制限に話せる安心感」）

```
トリガー: 5通目の返信直後（会話が盛り上がった瞬間）

構成:
  上部: 「もっと話を続けますか？」（責めない表現）

  中央: 感情訴求コピー（機能一覧は最小限）
  ╔══════════════════════════════╗
  ║  毎晩、話せる場所。           ║
  ║  ¥980 / 月                    ║
  ║  ────────────────             ║
  ║  ✓ 無制限チャット             ║
  ║  ✓ 週次レポート               ║
  ║  ✓ 感情カレンダー             ║
  ╚══════════════════════════════╝

  [7日間無料で試す] ← メインCTA
  いつでも解約できます（1タップ）← 小さく添える

  あとで（無料のまま続ける） ← 見えるが小さい

削除するもの:
× 年間プランの同時表示（混乱させる。月額で慣れてから）
× 法人プランへの導線（ユーザーが混乱する）
```

## 10. Awarefyユーザー獲得戦略（新設）

### Awarefyでよく見られる離脱パターン

競合調査・ユーザーレビュー分析から特定した「Awarefyをやめた理由」トップ5：

```
1位: 「機能多すぎて何をすれば良いかわからない」
2位: 「CBTのワークが義務的に感じて続かない」
3位: 「無料プランではAI機能がほぼ使えない」
4位: 「AIが同じような返答しかしない」
5位: 「感情の数値入力が苦手」
```

これらの離脱者は、**「メンタルケアに関心があり・アプリ課金経験もある」**という理想的な見込みユーザー。

### 獲得チャネル設計

**①App Store ASO（キーワード）**
```
狙うキーワード:
  「メンタルケア 話す」「AI 悩み 匿名」「愚痴 聞いてほしい」
  「メンタル アプリ 続かない」「AIカウンセリング 気軽」

あえて狙わないキーワード:
  「認知行動療法」「CBT」← Awarefyの縄張り。そこで戦わない
```

**②SNSコンテンツ設計（ビルドインパブリック）**
```
投稿テーマ案:
  「ワークをこなすんじゃなくて、ただ話せるアプリを作っています」
  「『続かない』という声を聞いて、機能をゼロから削り直しました」
  「AIに愚痴を言ったら返事が来た。それだけでいいと思って作りました」

→ Awarefyの「続かなかった」という感情に共鳴するコンテンツ
```

**③Awarefyユーザーへのリーチ**
```
Awarefyのアプリレビューを定期観測:
  「続かない」「ワークが続かなかった」というレビューに対して、
  App Storeの開発者返信で「こういう方向けのアプリも作りました」
  ← ただしApp Storeのガイドライン確認が必要

Twitter/Xの検索アラート:
  「Awarefy やめた」「Awarefy 続かない」
  → 同情・共感コメントから自然にhonneを知ってもらう
```

### Paywallでの「乗り換え訴求」追加案

```
現行: 「毎晩、話せる場所。¥980/月」

追加案（A/Bテスト対象）:
  「ワークなし。数値入力なし。ただ話すだけ。¥980/月」
```
