## 1. プロジェクト概要

### アプリ名
**honne**（ホンネ）

### キャッチコピー
> 裁かれない場所で、本音を話そう。

### プロダクト定義
匿名・無判断のAIに本音を吐き出し、自己理解を深めるメンタルウェルネスアプリ。

解決策やアドバイスは提供しない。「話して、少し楽になった」という体験だけを届ける。

### コアバリュー（優先順）

| 順位 | 価値 | 定義 |
|------|------|------|
| 1 | **無判断** | AIは絶対に批判・評価しない |
| 2 | **匿名** | 名前・メアドなしで即開始できる |
| 3 | **整理** | 話しながら気持ちが言語化される |
| 4 | **記録** | 感情の変化が可視化される |
| 5 | **無干渉** | 解決策・アドバイスを急かさない |

### ポジショニング
```
「Awarefyは続かなかった」
「日記では足りない」
「心療内科は大げさ」

→ その全員が向かえる白いスペース
```

## 3. 機能要件

### 3-1. Must Have（v1.0 必須）

#### F01: 匿名スタート
- アカウント登録なしで即開始する
- 起動から最初のAI返答まで、3タップ以内で到達できること
- デバイス固有のUUID（Supabase匿名認証）で識別する

#### F02: オンボーディング（3画面）
- スライド1: 問いかけ画面「夜、誰かに話を聞いてほしくなることはありますか。」
- スライド2: 安心の担保（名前不要・暗号化保存・批判しない）
- スライド3: 話題セレクター（仕事 / 人間関係 / 将来 / なんとなく）
- 初回起動のみ表示し、以降はホーム画面を表示する

#### F03: チャット画面（コア機能）
- AI（OpenAI GPT-4o）との1対1テキスト会話
- タイピングアニメーション（ドット3つが順次拡大）
- 新規メッセージ到着時の自動スクロール
- 入力テキストが空のとき、送信ボタンを非活性にする
- 入力欄は最大4行まで自動拡張する

#### F04: 話題セレクター
- 最初のAIメッセージ直下に横スクロールで表示する
- 「仕事」「人間関係」「将来」「なんとなく」の4択
- 選択後はフェードアウトして非表示にする
- 2回目以降のセッションでは表示しない

#### F05: 感情タグ自動付与
- セッション終了時にGPT-4oが感情タグをJSON出力する
- タグ候補: 「疲れ」「不安」「怒り」「悲しみ」「孤独」「プレッシャー」「混乱」「落ち着き」「前向き」「スッキリ」
- 最大3つを自動選択して端末に保存する
- 一言サマリーも同時に保存する

#### F06: セッション終了フロー
- ナビゲーションバーの「終わる」ボタンで確認ダイアログを表示する
- ダイアログ: 「今日はここまでにしますか？」→「終わる」/「続ける」
- 「終わる」選択後: 感情タグ確認画面を表示し、ホームへ遷移する

#### F07: ホーム画面
- 連続記録日数（ストリーク）をカード形式で表示する
- 今週の感情タグ（上位最大3つ）を表示する
- 「今日も話す」CTAボタンを常に最下部に表示する
- 話題クイック選択ボタン（4種）を表示する

#### F08: 無料/Pro制限
- Free プラン: 1日5メッセージまで
- 5通目のAI返答直後にPaywallを表示する
- Pro プラン: メッセージ数制限なし

#### F09: Paywall（課金画面）
- ボトムシートとしてチャット画面上に重ねて表示する
- 7日間無料トライアル付きの月額プランを提示する
- 「いつでも解約できます（1タップ）」を必ず表示する
- 「あとで（無料のまま続ける）」の選択肢を提供する

#### F10: セーフガード
- 危機ワード（「死にたい」「消えてしまいたい」「自分を傷つけ」等）を検知する
- 検知時は即座に専門機関の連絡先を返答する
- よりそいホットライン（0120-279-338）・いのちの電話（0570-783-556）を表示する

## 6. AI会話設計要件

### 6-1. AIの人格

- 名前を名乗らない（「私は〜」と言わない）
- 存在感: 「深夜に話を聞いてくれる、静かな誰か」
- 友達でも、カウンセラーでも、AIアシスタントでもない

### 6-2. 3ステップ会話モデル

```
[ステップ1: 共感フェーズ] 最初の2〜3往復
  目的: 「ここは安全だ」と感じてもらう
  行動: 相手の言葉をそのまま使って返す / 評価・判断・アドバイスをしない

    ↓ 移行条件: ユーザーが「うん」「そうかも」など肯定した後

[ステップ2: 深掘りフェーズ] 中盤の3〜5往復
  目的: 「本当に何が嫌なのか」を一緒に見つける
  行動: オープンクエスチョンで核心に近づく / 「事実」より「感情」を掘る

    ↓ 移行条件: ユーザーが「〜だと思う」など自分の解釈を語り始めた後

[ステップ3: 整理フェーズ] 最後の1〜2往復
  目的: 「話してよかった」と感じてもらう
  行動: 今日話してくれたことをひと言でまとめる / 解決策は出さない

    ↓ 移行条件: ユーザーが「楽になった」「ありがとう」のサインを示した後

[セッション終了]
  感情タグ付与 + 一言サマリー保存
```

### 6-3. 返答ルール

- **必ず1〜3文以内** で返す（長文禁止）
- 共感フェーズ最初の2往復は **1文のみ**
- 質問は **1つだけ**（2つ重ねない）
- 同じ語尾パターンを2回連続で使わない
- リスト形式（①②③）は使わない

### 6-4. 禁止事項（AI返答）

```
× 「〜すべきです」「〜した方がいい」
× 「わかります」「頑張ってください」
× 「それはあなたのせいではありません」（決めつけ）
× 転職・離婚・絶縁などの重大な決断を勧める
× 医療・法律アドバイス
× 「どうしたいですか？」（共感フェーズ中は特に厳禁）
× 解決策・提案（ユーザーが求めた場合のみ可）
```

### 6-5. 危機対応（セーフガード）

**レベル1（様子を見る）:**
トリガー: 「消えたい」「限界」「もう嫌だ」「死にそう（比喩）」
対応: 通常フローを続けながら信頼できる人の存在を確認する

**レベル2（即座に切り替え）:**
トリガー: 「死にたい」「消えてしまいたい」「自分を傷つけ」「もう終わりにしたい」
対応: 以下の固定文を返す（変更不可）

```
「話してくれてありがとうございます。
今、あなたのことがとても心配です。
一人で抱えないでください。

よりそいホットライン: 0120-279-338（24時間）
いのちの電話: 0570-783-556

今すぐ話を聞いてくれる人がいます。」
```

### 6-6. 感情タグ付与仕様（セッション終了時）

AI が以下の中から最大3つを選択し、JSON形式で返す：

```json
{
  "tags": ["疲れ", "孤独"],
  "summary": "仕事の人間関係について話しました"
}
```

候補タグ: `疲れ` `不安` `怒り` `悲しみ` `孤独` `プレッシャー` `混乱` `落ち着き` `前向き` `スッキリ`

## 7. データモデル

### 7-1. エンティティ定義

#### Message
```
id:        UUID
role:      "user" | "assistant"
content:   String
timestamp: Date
sessionId: UUID
```

#### Session
```
id:           UUID
userId:       UUID（匿名UUID）
startedAt:    Date
endedAt:      Date?
emotionTags:  [EmotionTag]   ← 最大3つ
summary:      String?        ← 一言サマリー
messageCount: Int
```

#### EmotionTag
```
enum: 疲れ / 不安 / 怒り / 悲しみ / 孤独 / プレッシャー / 混乱 / 落ち着き / 前向き / スッキリ
color: 各タグに対応するUIカラー
```

#### UserState
```
userId:                UUID
streak:                Int          ← 連続記録日数
lastSessionDate:       Date?
isPro:                 Bool
remainingFreeMessages: Int          ← 1日5回リセット
onboardingCompleted:   Bool
```

### 7-2. ローカル保存戦略

- 会話履歴: `UserDefaults` または `Core Data`（端末内暗号化）
- ユーザー状態: `UserDefaults`
- OpenAI APIキー: Keychain に保存（本番）

### 7-3. Supabase（バックエンド）

- **匿名認証**: `Anonymous Sign-in` でUUIDを発行する
- **会話履歴**: PostgreSQL に暗号化して保存する（Pro ユーザーのみ）
- **Edge Functions**: OpenAI API プロキシ（APIキーをクライアントに露出させない）

## 9. セキュリティ・プライバシー要件

### 9-1. データ保護

- 会話ログは端末内で暗号化して保存する
- サーバー側（Supabase）に保存する場合も暗号化する
- APIキーはクライアントコードに直接記述しない（Keychain / Edge Functions経由）
- ユーザーがセッションを削除した場合、サーバー側データも削除する

### 9-2. 匿名性の担保

- 名前・メールアドレスを収集しない
- Apple IDでの認証をオプションとする（強制しない）
- Supabase の匿名認証でUUIDのみ管理する

### 9-3. App Store プライバシー申告（必須）

- データの種類: 「ヘルスとフィットネス」「識別子（UUID）」「使用状況データ」
- サードパーティ共有: OpenAI（会話内容）・Firebase（クラッシュログ）を申告する
- プライバシーポリシーURL: リリース前に必ず作成・公開する

### 9-4. 医療的免責

- アプリ内に「医療機器ではありません」を明記する
- 「専門的な医療アドバイスを提供しません」を利用規約に記載する
- 危機時（レベル2）は必ず専門機関へ誘導し、会話を継続しない
