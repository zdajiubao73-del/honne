## Phase 0：課題検証（CPF）
**期間：2週間 / 今すぐ開始**

### 目標
「仮想インタビューの仮説を、実在する人間で確認する」

CPF_INTERVIEW.mdの仮想インタビューはあくまで仮説。
**実在する人間20人のインタビューなしに開発を始めてはいけない。**（起業の科学）

### やること

#### Week 1：インタビュー準備〜実施（10人）
```
Day 1-2: 候補者リストアップ
  ターゲット:
  ・一人暮らしの20〜30代の知人（さやか型）
  ・看護師・介護士・保育士の知人（ゆか型）
  ・フリーランサーの知人（翔太型）
  ・Twitterで「誰にも言えない」「吐き出したい」で検索してDM

Day 3-7: インタビュー実施（1人30分・オンライン可）
  設問（CPF_INTERVIEW.mdの6問を使用）:
  Q1. 最近、誰かに話したかったのに話せなかった経験はありますか？
  Q2. そのときどうしましたか？
  Q3. 今の方法の一番の不満は何ですか？
  Q4. 理想の方法は？
  Q5. AIに話しかけることへの抵抗は？
  Q6. 月980円なら？→「今日から使えたら登録しますか？」
```

#### Week 2：残り10人 + 分析
```
Day 8-12: インタビュー実施（追加10人）
Day 13-14: KJ法で分析
  ・「言動」→「本音」→「深層の本音」の3層で整理
  ・CPF_INTERVIEW.mdの仮説と比較
  ・ペルソナの修正点を書き出す
```

### CPF完了条件（起業の科学より）
```
□ 20人以上インタビュー完了
□ 「ただ聞いてほしい・整理したい」という課題が確認できた
□ 代替手段の不満が言語化できた
□ エバンジェリストカスタマー3〜5人確保（TestFlight協力者）
□ 想定と違った発見を3つ以上言語化した
```

### 必要なもの・コスト
```
コスト: ¥0（知人ならカフェ代のみ）
ツール: Googleフォーム（議事録）・Notion（分析）
時間:   1日2〜3人 × 2週間 = 約20〜30時間
```

## Phase 1：解決策検証（PSF）
**期間：2週間**

### 目標
「会話フローの設計が、実際のユーザーに刺さるかを確認する」

### やること

#### Week 3：ペーパープロト → ソリューションインタビュー
```
Day 1-2: ペーパープロトタイプ作成
  ・A4紙 or iPad手書きで4画面を描く
    ① オンボーディング画面
    ② 話題セレクター
    ③ チャット画面（AI返答サンプル付き）
    ④ セッション終了・感情タグ画面
  ・実際の会話サンプルをCONVERSATION_FLOW.mdから印刷

Day 3-5: CPF協力者5人に見せる
  「これで話してみたいと思いますか？」
  「最初の画面を見て、何のアプリかわかりますか？」
  「この会話の返答、どう感じましたか？」
```

#### Week 4：Figma → プロダクトインタビュー
```
Day 6-10: Figmaでインタラクティブプロト作成
  参考: REFINED.mdの画面設計（4画面）
  ・オンボーディング（3画面）
  ・チャット画面
  ・Paywall
  ・ホーム（ストリーク表示）

Day 11-14: 新規5人にプロダクトインタビュー
  「今、何をしようとしていますか？（操作中に聞く）」
  「このボタンは何をするものだと思いますか？」
  「この会話の返答を読んで、どう感じましたか？」
  「続けて使いたいと思いましたか？」
```

### PSF完了条件
```
□ 「この返答なら話したくなる」という反応が5人中3人以上
□ オンボーディングを見て「何のアプリか」が即座に伝わる
□ 「課題なし・ワークなし」設計への肯定的反応
□ Must-have機能が確定（変更しない）
□ 削除すべき機能が明確になった
```

### 必要なもの・コスト
```
コスト: Figma 無料プラン（個人利用）
時間:   約20〜30時間
成果物: インタラクティブプロト（TestFlight代わりの検証用）

## Phase 2：MVP実装
**期間：6〜8週間**

### 開発方針
```
「恥ずかしい状態でリリースしないなら、遅すぎる」— リード・ホフマン

完璧を目指さない。TestFlight 10人が使えれば十分。
Must-haveだけ実装。Should-haveは絶対に入れない。
```

### 技術スタック（PLAN.mdより）
```
フロントエンド:  Swift / SwiftUI（iOS 17以上）

AI:             OpenAI GPT-4o
課金:           RevenueCat
通知:           Firebase Cloud Messaging
クラッシュ解析: Firebase Crashlytics
```

### 実装ロードマップ

#### Week 5-6：コア実装（チャット画面 + OpenAI連携）
```
優先度1: チャット画面（アプリの95%の価値）
  ・SwiftUIでチャットUI
  ・メッセージバブル（AI / ユーザー）
  ・タイピングアニメーション（「.」→「..」→「...」）
  ・テキスト入力 + 送信ボタン

優先度2: OpenAI GPT-4o連携
  ・Supabase Edge FunctionsでAPIプロキシ
  ・CONVERSATION_FLOW.mdのシステムプロンプトを実装
  ・フェーズ管理ロジック（共感→深掘り→整理）
  ・セーフガード（危機ワード検知）

この2つだけでTestFlight配布が可能になる
```

#### Week 7-8：匿名認証 + 最低限の記録
```
優先度3: Supabase匿名認証
  ・名前・メアド不要でUUID発行
  ・会話履歴の暗号化保存

優先度4: オンボーディング（3画面）
  ・「夜、誰かに話を聞いてほしいとき」
  ・匿名の担保3点（名前不要・暗号化・批判しない）
  ・「はじめる」ボタン → 即チャット

優先度5: 感情タグ自動付与
  ・セッション終了時にGPTがタグをJSON出力
  ・端末側で保存（表示はv1.1以降でOK）
```

#### Week 9-10：RevenueCat + ホーム画面 + バグ修正
```
優先度6: RevenueCat課金実装
  ・Free（5メッセージ/日）
  ・Pro（¥980/月）
  ・7日間無料トライアル
  ・Paywall画面（REFINED.mdの設計通り）

優先度7: ホーム画面（最小限）

## Phase 3：TestFlight & PMF探索
**期間：4〜8週間（PMF確認まで継続）**

### 目標
「ショーン・エリス・テストで40%が『非常に残念』になるまで改善する」

### TestFlight配布（Week 11）
```
配布対象:
  ・Phase 0で確保したエバンジェリストカスタマー3〜5人（最優先）
  ・追加で知人・友人10〜20人
  ・合計10〜30人

伝え方:
  「荒削りなアプリです。壊れても大丈夫です。
   週1回のフィードバックをもらえると助かります」
  ← 起業の科学：「恥ずかしい状態で出す」
```

### 計測開始（Week 11〜）
```
AARRR指標を毎週計測:
  Activation:  オンボーディング完了 → 初回チャット完了率（目標: 80%）
  Retention:   7日後に再開したか（目標: 40%以上）← 最重要
  Revenue:     無料→Pro転換率（目標: 15%以上）
  Referral:    「友達に勧めたい」と言ったか

AHAモーメント仮説を検証:
  「初回セッション完了から48時間以内に2回目を開いたか」
  ← これが定着の予測指標（CPF_INTERVIEW.md仮説）
```

### ショーン・エリス・テスト
```
10人使ったタイミングで実施:
  「このアプリがなくなったらどう思いますか？」
  A. 非常に残念
  B. 少し残念
  C. 残念ではない（他の代替手段がある）

目標: Aが40%以上 = PMF達成と判断
現実: 最初は10〜20%。スプリントを繰り返して40%を目指す
```

### スプリント設計（2週間1サイクル）
```
Week N:   使ってもらう + 計測
Week N+1: インタビュー（3〜5人）+ 最も影響の大きい1点を改善
           ↓
           ・AIの返答品質が低い → システムプロンプト改善
           ・オンボーディングで離脱 → 画面を削る
           ・Paywallで離脱 → コピー変更
           ・7日後に来ない → プッシュ通知のタイミング調整

改善の優先順位:
  1位: AIの会話品質（返答の自然さ・フェーズ管理の精度）
  2位: オンボーディング完了率
  3位: 7日リテンション率
  4位: 課金転換率
  ※ 課金は最後。リテンションが先。
```

## Phase 4：App Store リリース
**期間：2〜3週間（PMF確認後）**

**PMF確認前にApp Storeに提出しない。**

### App Store準備チェックリスト

#### コンテンツ準備
```
□ アプリアイコン（REFINED.mdのB案：暗いグラデーション + 白い半円）
□ スクリーンショット 6〜10枚（iPhone 6.7インチ必須）
    1枚目: 「解決策は、いりません。」コピー + チャット画面
    2枚目: 会話サンプル（実際に近い会話）
    3枚目: 安心の担保3点（名前不要・暗号化・批判しない）
    4枚目: セッション終了後の感情タグ
    5枚目: ホーム（ストリーク表示）
□ プレビュー動画（30秒）
    実際の会話が流れる動画。「ただ話しかけるだけ」を見せる
□ App Store説明文（REFINED.mdのコピーを使用）
□ キーワード（100文字以内）
    「本音,匿名,AI,相談,悩み,吐き出し,メンタル,話す,聴く,感情」
```

#### 審査対策
```
□ 「医療機器ではない」「専門的な医療アドバイスを提供しない」を明記
□ 危機対応（よりそいホットライン誘導）の実装を確認
□ プライバシーポリシー作成（Supabase・OpenAI利用を明記）
□ 利用規約作成
□ 年齢制限: 12歳以上推奨（メンタルヘルスカテゴリ）
□ データ収集の申告（App Storeのプライバシー栄養表示）

審査期間: 通常1〜7日（初回は長め）
カテゴリ: ヘルスケア/フィットネス
```

#### 価格設定
```
本体: 無料
サブスク: ¥980/月（Pro）
年間: ¥7,800/年（Pro、月換算¥650）
無料トライアル: 7日間

注意: 年間プランはPaywallに同時表示しない
     月額で慣れてから、設定画面で見せる（REFINED.md方針）

```

  「ユーザーから来た声（匿名・了承済み）と、それへの回答」
  「AIの返答設計で悩んでいること」

目標:
  ・自分のスタートアップへの共感を集める

| リスク | 発生タイミング | 対策 |
