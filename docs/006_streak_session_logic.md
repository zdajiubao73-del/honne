# ストリーク・セッション管理ロジックの完成

## 概要
連続記録日数（ストリーク）の正確な計算と日次リセットロジックを実装する。「昨日も話した → ストリーク+1」「2日以上空いた → ストリーク0にリセット」を正確に処理する。

## 背景
`UserState.streak` は存在するが、計算ロジックが実装されているか未確認。REQUIREMENTS.md §5-2（Screen 3）でホームのストリークカードは主役要素として定義されており、数値の正確性がリテンション施策（連続記録の動機づけ）に直結する。
